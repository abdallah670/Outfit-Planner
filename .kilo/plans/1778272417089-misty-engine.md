# Backend Error Fixes - Comprehensive Plan

## Overview
Fix all compilation errors in the OutfitPlanner .NET backend. The solution has multiple projects with interface mismatches, missing imports, duplicate methods, and incorrect signatures.

---

## Errors Identified

### 1. **Program.cs - Missing Namespace Import** (CRITICAL)
- **Location**: `src/OutfitPlanner.Api/Program.cs` line 252
- **Issue**: `AccountUnlockBackgroundJob` is used but its namespace `OutfitPlanner.Infrastructure.Services` is not imported
- **Current**: `using OutfitPlanner.Infrastructure;` (line 14) does NOT include sub-namespaces
- **Fix**: Add `using OutfitPlanner.Infrastructure.Services;` to the using directives

### 2. **IGenericRepository.cs - Incorrect Method Signatures** (CRITICAL)
- **Location**: `src/OutfitPlanner.Application/Contracts/Persistence/IGenericRepository.cs`
- **Issues**:
  - Line 20: `SumAsync(Expression<Func<T, bool>> selector)` - wrong, should be CountAsync with predicate
  - Line 21: `AverageAsync(Expression<Func<T, bool>> selector)` - wrong, should use numeric selector
  - Line 24: `CountAsync(Expression<Func<T, decimal>> selector)` - wrong, Count should take bool predicate
- **Fix**: Replace with correct signatures:
  ```csharp
  Task<int> CountAsync(Expression<Func<T, bool>> predicate);
  Task<decimal> AverageAsync(Expression<Func<T, decimal>> selector);
  Task<decimal> SumAsync(Expression<Func<T, decimal>> selector);
  ```

### 3. **GenericRepository.cs - Duplicate/Incorrect Methods** (CRITICAL)
- **Location**: `src/OutfitPlanner.Persistence/Repositories/GenericRepository.cs`
- **Issues**:
  - Lines 82-85: `SumAsync(Expression<Func<T, int>> selector)` returns int - duplicate with different return type
  - Lines 97-100: `SumAsync(Expression<Func<T, decimal>> selector)` returns decimal - duplicate
  - Both exist alongside the interface's incorrect signatures causing mismatch
- **Fix**: Keep only the decimal version (as per proper interface), remove the int version

### 4. **Enums.cs - Typo in Enum Values** (MINOR)
- **Location**: `src/OutfitPlanner.Domain/Entities/Enums.cs`
- **Issues**:
  - Line 53: `Follwers` typo (should be `Followers`)
  - Line 118: `Follwers` typo in `Visibility` enum (should be `Followers`)
- **Fix**: Correct spelling to `Followers`

### 5. **ValidationException.cs - Missing Using** (MINOR)
- **Location**: `src/OutfitPlanner.Application/Exceptions/ValidationException.cs`
- **Issue**: Uses `ValidationResult` from FluentValidation but no `using FluentValidation;` or `using FluentValidation.Results;` at top
- **Current**: Only has `using FluentValidation.Results;`
- **Fix**: Verify the using is correct - `ValidationResult` is in `FluentValidation.Results` namespace, so it's actually correct. No fix needed.

### 6. **OutfitImageProcessingService.cs - Using Statement** (CHECK)
- **Location**: `src/OutfitPlanner.Infrastructure/Services/OutfitImageProcessingService.cs`
- **Issue**: Uses `OutfitLayoutConfig` class which is defined in `OutfitPlanner.Application.Contracts` namespace
- **Current**: Has `using OutfitPlanner.Application.Contracts;` (line 4) which should work since `OutfitLayoutConfig` is nested inside that namespace in the interface file
- **Status**: This is actually OK - `OutfitLayoutConfig` is defined in `IOutfitImageProcessingService.cs` in the same namespace

### 7. **Interface Implementation Mismatches** (POTENTIAL)
- Multiple repository interfaces and implementations need verification:
  - GenericRepository should implement IGenericrepository correctly
  - Check all repository interfaces don't have unimplemented methods
  - Verify IUnitOfWork implementation matches interface
