## URGENT FIX: Profile Page Blank Issue

### Problem
The profile page at `http://localhost:4200/profile` displays nothing. Root cause: all profile content is wrapped in an `<ng-template>` that's never rendered.

**File**: `src/outfit-planner-ui/src/app/presentation/pages/profile/profile.component.html`

**Bug** (lines 14-426):
```html
<ng-template #ownProfileTpl>
  <ng-container *ngIf="profile$ | async as profile">
    <!-- ALL PROFILE CONTENT -->
  </ng-container>
</ng-template>
```
The `ng-template` is inert — it defines a template but never renders it. The page only shows loading/error states, then blank.

### Solution
Remove the `<ng-template>` wrapper so the content renders directly:

**Change from**:
```html
<ng-template #ownProfileTpl>
  <ng-container *ngIf="profile$ | async as profile">
    ... content ...
  </ng-container>
</ng-template>
```

**To**:
```html
<ng-container *ngIf="profile$ | async as profile">
  ... content ...
</ng-container>
```

Optionally add empty state fallback:
```html
<ng-container *ngIf="profile$ | async as profile; else noProfile">
  ... content ...
</ng-container>
<ng-template #noProfile>
  <div *ngIf="!(loading$ | async) && !(error$ | async)" class="empty-state">
    <p>No profile data available</p>
    <button (click)="reload()">Reload</button>
  </div>
</ng-template>
```

This single change will make the profile page render correctly when data is loaded.

## Trade-offs / Considerations
- **Material vs Custom**: The new design removes Material components entirely in favor of custom HTML/CSS — acceptable for this design-focused change.
- **Poll Voting UX**: Reference shows poll voting inline within feed card; current PollCardComponent had separate voting UI. Streamlined approach: clicking an option immediately votes and shows results.
- **Accessibility**: Ensure buttons maintain proper ARIA labels and focus states.
- **Performance**: Grid layout with many posts; ensure images lazy-loaded (existing behavior likely OK).
- **Icon Dependency**: Using Iconify CDN (already referenced in design). Could install `@iconify/angular` package if preferred.
